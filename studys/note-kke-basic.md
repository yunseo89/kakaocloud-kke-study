## MSA(Micro Service Architecture)의 등장과 가상화 기술
1. 소프트웨어 개발의 진화 과정<br>
- 모놀리식 아키텍쳐(MA): 모든 요소(코드)를 하나의 어플리케이션에 구축해서 배포하는 방식. 형태가 단순해서 유지보수가 편리하지만 부분 장애가 전체 서비스의 장애로 확대될 수 있다. 
- 서비스 지향 아키텍쳐(SOA): 서비스를 중심으로 한 방식, 느슨한 결합을 지향
- 마이크로 서비스 아키텍쳐(MSA)(현재): 각 서비스를 독립적으로 개발, 배포, 운영할 수 있는 구조
  -> 서비스가 모두 분산되어 복잡할 수도..

2. 가상화 기술<br>
= 실제 리소스를 추상화하고, 여러 개의 가상환경을 생성하는 기술

- 가상 머신(Virtual Machine, VM) : 하나의 물리적 서버에서 여러 개의 독립적인 가상 서버를 운영할 수 있도록 하는 기술
- 컨테이너 가상화(containerization) : 응용 프로그램과 모든 종속성을 패키지화 해서 이식성을 향상시키고, 빠른 배포 및 확장이 가능한 기술
