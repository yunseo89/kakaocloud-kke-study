## 깃허브 관련 개념 정리
### 1. 마크다운(Markdown)이란?

> HTML로의 변환이 가능한 마크업 언어로, 특수기호와 문자를 이용한 간단한 문법을 사용한다.
> 
> 장점: 별도의 도구가 필요없고 텍스트로 저장되기 때문에 용량이 적다 // 단점: 표준이 없다.

**마크업 언어란?**

-> 문서나 데이터의 구조를 명시하기 위해 사용하는 컴퓨터 언어이다. 일반 텍스트에 태그를 달아 화면에 어떻게 표시되는지를 결정함.<br>
대표적인 예로 HTML, XML, 마크다운 등이 있다.

### 2. 마크다운 문법

1. 헤더는 # 사용
2. 띄어쓰기는 엔터 또는 태그 br 사용 
3. blockquote(인용문) 은 이메일에서 사용하는 > 블럭 인용문자를 사용, 이 안에서 다른 마크다운 요소를 포함할 수 있음
4. 순서있는 목록은 숫자와 점을 사용. 어떤 번호를 입력해도 순서는 내림차순으로 정의된다.
5. 순서 없는 목록은 글머리 기호로 *, +, - 지원
6. 코드는 하나의 탭으로 들여쓰기를 하면 시작, 들여쓰기를 않은 행을 만날때까지 변환이 계속된다.
7. 수평선은 * * *, ***, ***** 다 가능, 보통 페이지 나누기 용도로 많이 사용한다.
8. 링크달기: 그냥 달면 됨. <https://docs.kakaocloud.com/tutorial/container/cr-helm-k8se>
9. 강조는 **이걸로->***

11. 줄바꿈은 3칸 이상의 띄어쓰기가 필요

### 1. Github 기본 용어 정리
1. CLI vs GUI
   - CLI(command line interface)로 주로 터미널을 통해 컴퓨터와 상호작용하는 방식<br>
   - GUI(Graphical User Interface) 는 그래픽,, UI

3. Branch(브랜치) = 레포지토리에서 독립적으로 작업을 하기 위한 공간
4. Commit = 소스 코드의 업데이트를 확정하는 것. 확정된 순간의 코드 상태를 메세지와 함께 레포에 저장한다.<br>
-> 로컬 저장소에는 반영이 되지만 원격 저장소에는 반영되지 않은 상태.. 로컬: 내 컴퓨터, 원격(리모트): 원격 저장소

5. pull/push
   - pull: 원격저장소의 내용을 로컬로 끌어오는 것
   - push: 커밋한 내용을 원격 저장소에 업로드하는 것
